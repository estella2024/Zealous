# Zealous Project Rules

## Collaboration

- Default to a Lite workflow for this project.
- When communicating progress or requesting visual feedback, proactively provide a clear browser-viewable address the user can open in the built-in browser.
- Follow the simplest-path rule for those addresses: prefer the existing local app URL such as `http://localhost:3000/`, and only introduce a separate HTML entry or extra preview route when it is genuinely necessary.
- Do not build non-essential preview artifacts just to communicate changes.

## Ownership

- Stable project entry document: `README.md`
- AI-facing collaboration rules: `AGENTS.md`
- Continuous development log: add only when the project starts changing across multiple phases
- Phase handoff summary: add only when a real phase boundary appears
